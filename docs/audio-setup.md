# Подключение и настройка звука (колонка) на банке

Как вывести звук плеера на колонку через зелёный 3.5мм разъём. Воспроизводимо на
любой новой банке. Звуковой стек — **PipeWire** (`wpctl`).

## 1. Физика
- Колонку (mini-jack 3.5мм) — в **зелёный** разъём задней панели = **line-out** (выход).
  - розовый = микрофон (вход), синий = line-in (вход) — НЕ сюда.
- Включить питание колонки, выкрутить её собственную громкость.
- HDMI-аудио (через видеокарту, на LED-экран) НЕ используем.

## 2. Сессия
Команды — в графической сессии пользователя `cannect`. Если зашёл по SSH
(другая сессия), сначала укажи сокет PipeWire:
```bash
export XDG_RUNTIME_DIR=/run/user/1000
```

## 3. Выбрать правильный выход (аналог, не HDMI)
```bash
wpctl status
```
В разделе **Sinks** два варианта:
- `Built-in Audio Аналоговый стерео` ← зелёный разъём (нужный)
- `... (HDMI)` ← в экран через видеокарту (не нужен)

Если `*` стоит не на аналоговом — сделать его дефолтным (номер из списка, напр. 70):
```bash
wpctl set-default 70
```

## 4. Громкость и unmute
```bash
wpctl set-mute   @DEFAULT_AUDIO_SINK@ 0      # снять mute
wpctl set-volume @DEFAULT_AUDIO_SINK@ 1.0    # 100%
wpctl get-volume @DEFAULT_AUDIO_SINK@        # проверить
```
Выше `1.0` не задирать — PipeWire начнёт искажать; громче — ручкой на колонке.

## 5. Проверка
```bash
pw-play /usr/share/sounds/alsa/Front_Center.wav   # скажет «Front Center» из колонки
```

## 6. Персистентность
PipeWire/wireplumber сохраняет выбор выхода и громкость
(`~/.local/state/wireplumber`) → после перезагрузки звук останется на аналоговом
выходе и на той же громкости. Плеер (Chromium-поток) автоматически идёт в
дефолтный выход — отдельной настройки не требует.

## Если тишина — чек-лист
1. Колонка включена, её громкость не на нуле.
2. Правильный (зелёный) разъём.
3. `wpctl status` → дефолт (`*`) на «Аналоговый стерео», а не HDMI.
4. Sink не muted (`wpctl get-volume @DEFAULT_AUDIO_SINK@` без `[MUTED]`).
5. Из SSH — выставлен `XDG_RUNTIME_DIR=/run/user/1000`.
