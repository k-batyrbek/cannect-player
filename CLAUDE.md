# cannect-player — нативный Linux-плеер для LED-вендинг «банок»

> Этот файл — контекст-хендофф для Claude Code. Прочитай его целиком: он
> описывает что мы строим, почему, архитектуру и контракты сервера.

## Что это
Десктоп-приложение под **Ubuntu Linux** (ставится на мини-ПК внутри «банки» —
цилиндрической LED-вендинг-машины). Проигрывает рекламный плейлист на весь
экран (LED-поверхность банки), офлайн-устойчиво, со звуком, и управляется
удалённо. Заменяет текущий костыль (Chromium-киоск, открывающий
`https://cannect.kz/vending/<id>` во вкладке браузера).

## Зачем (проблемы текущего Chromium-киоска)
1. **Звук не играет** — браузерный Chromium обязан требовать жест юзера для
   autoplay со звуком. В **Electron это снимается флагом**
   `app.commandLine.appendSwitch('autoplay-policy', 'no-user-gesture-required')`
   → звук играет сразу. ЭТО ГЛАВНАЯ ПРИЧИНА перейти на Electron.
   (Проверено ffprobe: у видео есть нормальная аудиодорожка ~-17dB, дело было в браузере.)
2. **Нет офлайна** — при обрыве сети плейбэк рушится. Нужен локальный кэш видео.
3. **Переходы лагали** на первом, незакэшированном проигрывании (висел последний кадр).
4. **Обновления = выезд на точку** жать F5. Нужна удалёнка + авто-апдейт.

## Стек (решение)
- **Electron** (renderer = плеер-UI на React, main = Node-логика). Пользователь
  хочет именно полноценное приложение, не headless-демон.
- Звук: autoplay-policy off (см. выше) + выбор аудио-устройства.
- Видео: **локальный кэш** (качать на диск, играть с `file://`) → офлайн + плавно.
- Автозапуск: systemd + kiosk-fullscreen.
- (Опционально на будущее: mpv через IPC, если Chromium-рендер видео будет жрать CPU.)

## Что приложение делает (флоу)
1. Конфиг станции: `STATION_ID` (напр. ALM-002 = `6a2699575a677a6355883ea2`).
2. Опрашивает **`GET https://cannect.kz/api/stations/<STATION_ID>/queue`** →
   плейлист. Ответ: `data.queue[]` с полями `videoId, videoTitle, videoUrl,
   duration, aspectRatio ('16:9'|'9:16'), source ('smb'|'agency'), campaignId,
   bookingId`, плюс `data.operating` (в рабочих ли часах) и `data.timezone`.
3. **Кэширует** каждый `videoUrl` на диск (по хэшу URL), играет из кэша.
4. Играет на весь экран, **зациклено**:
   - `9:16` (portrait) → **4 вертикальные полосы** + чёрные разделители ~20px
     (так свёрстана реальная поверхность банки). Звук — только у 1-й полосы.
   - `16:9` (landscape) → одно видео на весь экран.
   - Плавный переход: следующий клип должен быть прогрет (кэш) до конца текущего.
5. **Репорт показа**: `POST /api/stations/<id>/report`
   `{ videoId, campaignId|bookingId, timestamp, duration, completed, billable }`.
   - `billable=false` для бонус-петель (повторы сверх забронированных showsPerHour).
   - Сервер САМ занулит billable вне рабочих часов (24/7-показ, биллинг 9–22) — не наша забота.
6. **Текущий плейбэк**: `POST /api/stations/<id>/current-playback`
   `{ event:'playback_started'|'playback_changed'|'playback_ended', campaignId,
   bookingId, videoId, videoTitle, expectedDuration, isDefault, startedAt }`.
   Нужно для атрибуции CV-аналитики к кампании (сервер связывает камеры с тем,
   что играет сейчас).

## Удалённое управление (чтобы не ездить на точку)
- **Tailscale** на каждой банке + на Mac разработчика → mesh-VPN, SSH сквозь
  NAT/венью-WiFi без белого IP. `ssh banka-smartpoint`.
- **Авто-апдейт приложения**: electron-updater (раздавать релизы с GitHub
  Releases или своего VPS).
- **Контент** обновляется сам (приложение опрашивает `/queue`) — без выездов.

## Что НЕ трогаем
- Сервер **cannect-web** не меняем — плеер только ПОТРЕБЛЯЕТ существующие API.
  (cannect-web: GitLab `cannect3/cannect-web` — прод + CI, GitHub
  `Batrbekk/cannect` — зеркало; прод `https://cannect.kz`.)
- CV-камеры — отдельный проект `cannect-camera` (не сюда).

## Откуда брать эталон UI
Логика плеера уже есть в вебе — порт из cannect-web:
`src/app/(vending)/vending/[id]/page.tsx` (4 полосы для 9:16, разделители,
буфер-гейт перехода ~6с, репорт-флоу, current-playback). 3D-превью банки НЕ нужно
(это для дашборда-конфигуратора, не для плеера).
